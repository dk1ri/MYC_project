MYC, Reserved and Mandatory Commands / Tokens

 

Author: DK1RI

Version V01.10

 

This paper describes reserved command tokens.

For usage, handling of commands see also [1] and [2]

 

Definitions and formats

 

HI   human Interface device, device-type h

CR   command-router, device-type c

LD   logic-device, device-type l

PR   program with MYC Interface or interface to program, device-type p

RU   rules device, device-type r

VA   device variable announcelist, device-type v

<c>   unique (per device) command token; one number (Byte, word long or x(8 Byte)). 0 (one byte always) is reserved for the basic

   announcement. &HxxF0 - &HxxFF are also reserved

<dt>   device-type: m, p, c, l, h, v, r

<dn>   device name, unique for a software; combination of MANUFACTURER DEVICEDESCRIPTION and VERSION

<ct>   command-type

<p>   property (parameter) of the type <n> or <s> in command and info

<pa>   property of the type <ty> or <sm> or <OPTION> in an announcement

<OPTION>  optional property

<des>    a string <sm>  used in announcements for identification or description of  commands and properties.

   <des> are optional sometimes. They can be used better human reading and for informative display of the HI and the LD.

   They are also used for the translation to real values.

<n>   numeric value, binary in commands, decimal readable in announcements

   For the selection of the command-format it is important, how many different values are needed. So a range of -3 to +200 has

   204 values and a byte with the range of 0 to 203 is used. A range from 1,0 to 2.0 needs 11 values from 0 to 10 – a byte is also

    sufficient.

   The human interface knows the real values by the readable and real values of <des> of the announcement  and will translate and

   display the real values.

   All sequence numbers are “0” based.

<s>   string, all characters. The string is preceded by the length; as byte, word or long, depending on maximum length.

   This type exists in commands and infos

<sm>   string (all characters except  , ;) Used in announcements. So for decimal ", comma" . (dot) must be used.

<l>   maximum length of a string <s> or <sm> in announcements (replaced by the number of bytes)

<ty>   a : bit, b : byte, i : integer, w - word, L - long, s - single, d - double, t - date. c : command-token,  <l> : string of type <s>

   t is the date format. The UNIX 64 bit format is used

   A bit is transferred as a byte with the value of &H00 or &H01

<default>  a default value in an announcement

<fixed>  a fixed informative value in an announcement

<announce>  complete announcement or announcement line, a string <sm>; the answer of 0 or 0FxxF0 command. is transmitted in <s> format

<data>   <s> or <n> in commands or info

separators  For the announcement parser (CR, HI or LD eg):  , ; (comma, semicolon)

   , is the separator between <pa> and <des>.

   ; is the separator between <pa>

&HxxFF  means highest number in 1, 2, or 4 Byte hex format

 

Every MYC device should understand mandatory reserved command tokens.

Reserved tokens are  (one byte) &H00 and &HxxF0 -&HxxFF.

Reserved one byte token &H00 must be the first token of the announcelist; the other reserved token must be at the end in the numerical sequence.

 

Six of these are used and defined now: &H00, &HxxF0 and &HxxFC - &HxxFF

These tokens are mandatory for all devices

 

Announcements Command &H00 and &HxxF0

 

All MYC devices should understand these commands.

 

&H00 basic announcement:

The basic announcement is called with the one byte command &H00.

announce: 0;<dt>;MANUFACTURER;DEVICEDESCRIPTION;VERSION;NUMBER_OF_DEVICE;LINELENGTH; NUMBER_OF_LINES

command: &H00

answer: <basic announcement>  as string <s>    

info:  &H00<basic_announcement>  as string <s>

 

The announcement is fixed and therefore format description is not transmitted. Instead the line with content is transmitted.

The first byte of the answer and the second of info is the one byte string-length. The &H00 announcement is limited therefore to 255 Bytes.

At startup or when the announcementlist changes the CR will send the &H00 command to devices, which can send commands. So they will know about a new list. The answer must be a valid string, but will not used by the CR.

 

  <dt> is one of the device-types:

      sending commands T: translated; N: not translated; -: not sending

       receiving commands  T: translated; N: not translated; -: not receiving

        receiving own reserved commands  T: translated; N: not translated; -: not receiving

         sending info T: translated; N: not translated; -: not sending

          receiving info T: translated; N: not translated; -: not receiving

           number of commands F: fixed V variable -: no commands

            number of rules F: fixed  V: variable  -: no rules

 

  c  command-router,  

   to lower level  N - - - N - - 

   to higher level  - N N N  - V V

   to m   N - - - N - -

   to p   T T N T T V V

   to l   T T - T T V V

   to h   T T N T T V V

   to v

   to r   N N N T T V V

  m normal device  - N N N - F F

  p programs:   T T T T T F -

  l logic-device;                  T        T        T        T        T        F        -

  h Human Interface  T - T T T F -

  v variable commands   - N N N - V V

  r rules device   - N N N - F V

 

MANUFACTURER DEVICEDESCRIPTION and VERSION are combined to a devicename are used in file names, so not all character are allowed. Preferred is a-z, A-Z, 0-9 _  

For a normal device  NUMBER_OF_DEVICES is 1, This value (all active devices) is important for the command-router as info for a

higher level CR

LINELENGTH is the maximum linelength of the announcement lines.

NUMBER_OF_LINES is the number of announcement lines including the basic announcement &H00 but without the reserved commands and rules. Using this number the requesting device will calculate the number of bytes used for the commandtoken

 

&HxxF0:

announce: 240;am,ANNOUNCEMENTS,<des>...<des>;<l>;<number_of_lines>    240 for 1 byte command-token

command: &HxxF0<n>            ask announcement line n

answer: <s>             announce line n

info:  &HxxF0<announce line 0>

 

<n> is a positional parameter of the memory, the commandtoken of the answer may be different.

<l> is the same as for the basic announcement; it is the length of the string of the memory.

< number_of_lines> is the number of the announcement lines including those for the reserved commands and rules.

 

Reserved numbers for <n> are:

n=&HxxF0- &HxxFB not used yet

n=&HxxFC rules announcements of a device only

n=&HxxFE  results in all announcement lines of a device.

n=&HxxFF is the same as n=0 for normal devices. The command-router will deliver the basic announcement lines of all attached and active devices as for the &H00 command of the device, but with translated command-token.

For <number_of_lines> > 240 two bytes must be used. The rule for selecting the paramter length is the same as for commandtoken.

 

Any answer of a &HxxF0 will have one  ANNOUNCEMENTS line only. So the commandrouter will drop the  ANNOUNCEMENTS lines of the other devices when building the complete list. The command-router answers all announcement commands by itself and will not forward these commands. So the asking devices will have the correct (translated) command-token.

Devices may be added or may disappear. It is assumed, that this will rarely happen, so that the CR will build a new list in this case and distribute it.

Future versions of the CR may use metacommands to handle disappeared devices.

 

Log on Command &HxxFB

 

This command is used by the commandrouter and rules-device only. If the system requires a log on for users to allow all or a subset of commands the user will be able to see the local commands of the commandrouter and only the local subset of the &HxxF0 command.  The log on data  are sent to the rulesdevice, wich will then sent  a set of “Q” rules to the commandrouter.

announce: 251;oa,LOGON[,<des>];<b>,mode;<s>,name;<s>,password

mode: 0 log out

 1 log on

 2 switch on user management (admin only)

 3 switch off user management (admin only)

 

Last Error Command &HxxFC

 

This command delivers the last error the device detected.

The command announcement has the first description  "LAST ERROR" to identify them in a list with the translated command-token, which is sent by the command-router.

announce: 252;aa,LAST ERROR[,<des>];<s>,last_error

command &HxxFC

answer  <s>

The format of the string is not defined. It should be used for debugging

 

Info Command &HxxFD

 

Some devices cannot send infos by themselves, devices with I2C Interface eg.

These devices can store infos in a info buffer and the command-router can poll with the info command.

A &HxxFD command will have one info. Different kind of infos <OPTION>  are defined as first description. Multiple <OPTION> are addressed by position.

The command announcement has the first description  "MYC INFO" to identify them in a list with the translated command-token, which is sent by the command-router.

 

announce: 253;aa,MYC INFO[,<des>]...[,<des>];<ty>[,<OPTION>,<des>

command: &HxxFD

Answer <s>|<n> 

info:

 

<OPTION>   Values:

b,ACTIVE   03  busy

    04  ready and active

    other <n> try again in <n> seconds 

b,ANNOUNCELIST  <n>  (new) list <n> available

    

more  to be defined

 

Individualization Command &HxxFE

 

The individualization commands read and write the individual properties of a device as names and number for different devices of the same device-name in a system and also properties of the communication with the command-router.

The command announcements have the first description  "INDIVIDUALISATION to identify them in a list with the translated command-token, which is sent by the command-router.

These properties usually are not written during normal usage but can be written for individualization before the normal usage.

At start up the CR must know the individual communication properties for the devices, which therefore must be stored locally,  if the device cannot send them by info (in an I2C environment eg).

 

announce: 254;oa,INDIVIDUALISATION[,des];<ty>,<OPTION>,<DEFAULT>[,<des>].. 

command 254<n><data>       <n> is the sequence number of the OPTION (0 based)

answer  -

info:  -

 

announce: 255;aa,INDIVIDUALISATION[,<des>];<ty>,<OPTION><DEFAULT>[,<des>]...

command 255<n>        <n> is the sequence number of the option (0 based)

answer  <data>

info:  255<data><data>       complete announcement line.

 

List of device-property token and values; (This list is not complete and consequently it is possible, that somewhen some OPTION are not understood.

The Optionnames mus be given in the announcement.

For multiple interfaces OPTIONs  for each interface must be grouped together and start with INTERFACETYPE.

A <DEFAULT> is mandatory where specified.

 

<OPTION>    Values:   comment:

<l>,NAME,<defaultname>     actual name, mandatory write/read

<n>,NUMBER<defaultnumber>    actual number, mandatory write/read

<l>,INTERFACETYPE,<fixed> I2C, RS232; Ethernet <fixed> is one of the values. This value is for information only and cannot be changed,

     CAN,RC5, RC6;USB Multiple Interfaces start with INTERFACETYPE, followed by the other parameters.

     Terminal  usually for CR only, interface, from where the program is called, no other options.

<ty>,ADRESS,<default>  COM Number /auto RS232

     Adressnumber (Byte) I2C

     <s>   Ethernet for devices

<t>,IF_PARAMETER1,<default> 19600Bd  for RS232

     100 /400  I2C speed, auto is default

     Port   Ethernet

<t>,IF_PARAMETER2,<default> like 8n1  RS232

TIMEOUT

ROUTING

USER     <s>   The user, who actually uses the interface. The LD can use this, to implement a right

        management by blocking the basic announcement of the interface

 

A normal device answers the &HxxF0&HxxFF announcement command with the default announcement and the &HFF command with the individual values.

The commandrouter must know the access to the device and therefore the complete individual data at start. It will forward the individual line so that asking devices will also have the individual name and number and not the default data.          .

In a hierarchical system an announcement answer of the commandrouter would be (tabs for clarification only):

<c>;c;...

 <c><dt>...

 ...

 <c>;oa, INDIVIDUALISATION...

 <c>;aa, INDIVIDUALISATION...

 ...

 <c>;c;...

  <c><dt>...

  ...

  <c>;oa, INDIVIDUALISATION...

  <c>;aa, INDIVIDUALISATION...

  ...

 ...

 <c>;oa, INDIVIDUALISATION...

 <c>;aa, INDIVIDUALISATION...

 ...

...

<c>;oa, INDIVIDUALISATION...

<c>;aa, INDIVIDUALISATION...

 

So identical devices with identical individualization can be identified by the nesting level and proper labeling can be done.
